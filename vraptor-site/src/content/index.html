Why VRaptor?

VRaptor is a refactor-friendly web framework where the authors try to implement RESTFul support.
Why would you use it rather than A, B or C?
VRaptor allows you to programmatic configure routes, validation rules and more in such a way that refactor does not break compatibility with the uris which identify your resources.

In the future, vraptor might be fully (level 3) RESTFul compatbile as it has been called, with support to hypermedia links, although this can be currently implemented by vraptor users in their own projects using the functionalities provided by VRaptor.
